Server і FastMCP

Два рівні абстракції в MCP Python SDK

Server і FastMCP

Два підходи

У MCP Python SDK є два способи створити сервер: низькорівневий Server і високорівневий FastMCP.

Головна різниця — рівень абстракції: скільки ви контролюєте самі, а скільки SDK робить за вас.

Server і FastMCP

Server — низький рівень

Базовий клас офіційного SDK. Дає прямий доступ до протоколу — ви збираєте сервер вручну.

  • Handlers вручну: @server.list_tools(), @server.call_tool()
  • Повний контроль: схеми, помилки, lifecycle, трасування
  • Більше boilerplate — але немає «чорної скриньки»
  • Debug простіший: видно кожен крок протоколу

Server і FastMCP

FastMCP — високий рівень

Фреймворк поверх MCP: пишете Python-функцію — FastMCP сам будує схему, валідацію й документацію.

  • Декоратори @tool, @resource, @prompt замість handlers
  • Менше коду, швидший старт — де-факто стандарт спільноти
  • Менше прозорості: внутрішня механіка як «чорна скринька»

Server і FastMCP

Коли що обирати

  • FastMCP — сервер з нуля, stdio або Streamable HTTP, швидкий старт. На курсі — основний шлях.
  • Server — потрібен повний контроль над протоколом, кастомна логіка або debug на рівні специфікації.

Server і FastMCP

Приклад: MCP Server

from mcp.server import Server
from mcp.server.stdio import stdio_server
import mcp.types as types

app = Server("my-server")

@app.list_tools()
async def list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="add",
            description="Додати два числа",
            inputSchema={
                "type": "object",
                "properties": {
                    "a": {"type": "integer"},
                    "b": {"type": "integer"},
                },
                "required": ["a", "b"],
            },
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "add":
        return [types.TextContent(type="text", text=str(arguments["a"] + arguments["b"]))]
    raise ValueError(f"Unknown tool: {name}")

async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write, app.create_initialization_options())

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())

Server і FastMCP

Приклад: FastMCP

from mcp.server.fastmcp import FastMCP

mcp = FastMCP()

@mcp.tool()
def add(a: int, b: int) -> int:
    """Додати два числа."""
    return a + b

if __name__ == "__main__":
    mcp.run(transport="stdio")

Той самий tool add — значно менше коду. На курсі далі працюємо з FastMCP.

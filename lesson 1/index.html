Агенти надсилають повідомлення, працюють з email, базами даних і API. Для цього потрібен кастомний код і обгортка у вигляді tools для кожного сервісу.

Основна проблема

  • Інтеграції жорстко привʼязані до одного середовища (наприклад, Cursor)

  • Кожен новий агент = нова інтеграція

Введення до MCP

Рішення: MCP

Model Context Protocol додає рівень абстракції — єдиний стандарт між агентами, tools і зовнішніми сервісами.

Інтеграція один раз → MCP-сервер → будь-який агент з підтримкою MCP.

Практичні сценарії

MCP у повсякденній роботі

Протокол використовують не лише розробники — будь-який користувач AI-інструмента може підключити зовнішні сервіси через MCP-сервери і працювати з ними в одному чаті.

Розглянемо три типові ситуації: особистий асистент, design-to-code і корпоративна аналітика.

Практичні сценарії

Персональний асистент

AI-помічник у Cursor або Claude Desktop бачить календар, нотатки в Notion і список задач — одночасно, без перемикання між вкладками.

Ви пишете «створи зустріч на завтра» або «додай нотатку до проєкту» — агент сам обирає потрібний tool і виконує дію через MCP.

Практичні сценарії

Design-to-code

Дизайнер передає макет у Figma. Розробник просить AI згенерувати компонент «як на макеті» — без ручного копіювання розмірів і кольорів.

Figma MCP дає агенту доступ до структури файлу: компоненти, токени, відступи. Код виходить набагато ближчим до дизайну, ніж генерація «на око».

Практичні сценарії

Enterprise-аналітика

У компанії дані розкидані по CRM, warehouse і внутрішніх API. Менеджер запитує в чаті: «Скільки угод закрили минулого кварталу?» — без SQL і без пʼяти вкладок.

MCP-сервери підключають системи до корпоративного асистента. IT контролює доступ на рівні серверів — без переписування інтеграцій під кожен новий AI-інструмент.

Практичні сценарії

Спробувати в Cursor

Щоб підключити MCP-сервер, додайте його в налаштування Cursor — наприклад, Notion MCP через поле mcpServers у конфігурації.

Після збереження агент отримує tools для пошуку, читання та створення сторінок у Notion. GitLab, Figma та інші сервери підключаються за тим самим принципом.

Фінал заняття

Підсумки

Без MCP кожен AI-агент потребує окремих інтеграцій із зовнішніми сервісами — це дорого, складно підтримувати і не масштабується.

MCP вирішує це через один сервер: інтеграція один раз, а підключитися може будь-який агент. Чим більше серверів і агентів — тим сильніший мережевий ефект.

У наступних лекціях розберемо, що таке MCP, його архітектуру та як побудувати власний MCP-сервер.

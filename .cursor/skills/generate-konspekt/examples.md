	MCP уже сьогодні використовується не лише розробниками...

	Уявіть AI-помічника, який бачить календар, Notion і задачі в одному чаті.
	Замість окремих інтеграцій — MCP-сервери. Агент обирає tool: «створи зустріч», «додай нотатку».
	Користувач описує результат мовою — агент виконує через MCP.
	## Design-to-code
	...
	## Enterprise-аналітика
	...
	## Підключення в Cursor
	Пояснення + json config + що отримає агент після підключення.

**Презентація:** для H2 «Як це працює» — копіювати HTML-діаграму з [`notion-to-presentation/diagram-examples.md`](../notion-to-presentation/diagram-examples.md).

## Що НЕ робити

- H4 або глибше
- Сухі однорядкові сценарії («Агент працює з календарем» — замало)
- H1 «Що таке MCP» / «Архітектура MCP» у Лекції 1
- Секції «Джерело» або meta про генерацію
- PNG/`<img>` для схеми «Як це працює» у презентації (лише `.mcp-flow`)

# Генератор конспекту MCP → Notion

## Мета

Створити або оновити сторінку **«Конспект»** для лекції курсу «Повний курс з MCP» у Notion, дотримуючись стандартів Hillel MAX.

## Вхідні дані

1. Номер лекції або тема (наприклад, «Лекція 1: Введення в MCP»).
2. План курсу з Notion (`content/notion-ids.json` → `plan`) або URL.
3. Опціонально: тезиси користувача в чаті.

## Вихід

- Сторінка «Конспект» у Notion під відповідною лекцією.
- ID сторінок — у [`content/notion-ids.json`](../../content/notion-ids.json).

## Алгоритм

1. Прочитати `content/notion-ids.json` для ID лекції та плану.
2. Отримати план курсу через Notion MCP (`notion-fetch` на `plan`).
3. Згенерувати конспект у Notion-flavored Markdown (специфікація: `notion://docs/enhanced-markdown-spec`).
4. Опублікувати через `notion-update-page` (якщо сторінка існує) або `notion-create-pages` (якщо нова).

## Структура конспекту (обовʼязково)

### Ієрархія заголовків

| Рівень | Синтаксис Notion | Призначення |
|--------|------------------|-------------|
| H1 | `# Назва {toggle="true"}` | Глави (блоки) — згортаються |
| H2 | `## Підтема` | Уточнення всередині глави |
| H3 | `### Деталь` | Максимальна глибина — **не глибше** |
| Фінал | `# Висновки {toggle="true"}` | Обовʼязковий блок наприкінці |

**Вкладеність:** max H3. Max 2 рівні вкладеності toggle/списків.

### Формат toggle-глав

Контент всередині H1 toggle має бути з **таб-відступом**:

```markdown
# Що таке MCP {toggle="true"}
	Короткий вступний абзац з ключовими термінами.
	## Model Context Protocol
	Пояснення поняття...
	- Пункт списку
	- Інший пункт
```

### Обовʼязкові секції лекції

1. **Вступ** (H1) — контекст і ключові поняття.
2. **Основні теми** (1+ H1 на кожну велику тему з плану).
3. **Висновки** (H1) — загальний підсумок заняття + місток до наступної лекції (див. шаблон нижче).

### Шаблон «Висновки»

**Не повторювати** деталі, схеми та покрокові flow з глав. Формат — 2–3 короткі абзаци:

1. **Що вивчили:** «На цій лекції ми розібрали / вивчили…» — узагальнення тем H1-блоків одним-двома реченнями.
2. **Навіщо це:** загальний сенс теми для студента (1 речення).
3. **Далі:** «У наступній лекції…» — 1 речення про наступну тему з плану курсу.

Приклад (Лекція 3):

```markdown
# Висновки {toggle="true"}
	На цій лекції ми розібрали архітектуру **MCP**: ролі Host, Client і Server, основу протоколу та три інтерфейси сервера — tools, resources і prompts.
	Також побачили, як виглядає **MCP Flow** — взаємодія застосунку, LLM і MCP Server під час виконання запиту користувача.
	У наступній лекції перейдемо до **transport layer** і створення власного MCP-сервера на Python.
```

## Правила стилю (Hillel)

Еталони оформлення (лише для агента, **не вставляти в конспект**):
- [Конспект AI-курсу](https://app.notion.com/p/27230ef38f2681a48d9ef52c3a3d828f)
- [Вимоги до конспектів](https://app.notion.com/p/710d48c848384cbb8aa9a428d0f28226)

**Заборонено в контенті для студентів:** секції «Джерело», посилання на зовнішню документацію як джерело, згадки workspace/репозиторію/підключених серверів, мета про генерацію контенту. Див. `.cursor/rules/mcp-course.mdc`.

- **Мова:** українська.
- **Речення:** короткі, зрозумілі, без «стін тексту».
- **Bold:** лише для ключових термінів (MCP, Host, Client, Server, Tool тощо).
- **Списки:** не робити жирним; не вкладати списки в списки.
- **Практичні сценарії:** вступний абзац + кожен H2 мінімум 2–3 речення (ситуація → роль MCP → результат). Не стискати до одного рядка.
- **Код:** fenced blocks з коментарями для MCP-прикладів.
- **H2 «Як це працює»:** текст у конспекті + позначка, що презентація використовує HTML-діagram `.mcp-flow` (див. `notion-to-presentation/diagram-examples.md`).
- **Зображення в Notion:** опціонально; для презентації — **не** PNG, лише кодована діаграма.
- **Інструкції:** покрокові для IDE, npm, Cursor MCP config.

## Маппинг план → лекції

| Модуль плану | Лекція | Теми H1 |
|---|---|---|
| Введення в MCP | 1–2 | Лекція 1: проблема/рішення MCP; Лекція 2: підключення готових серверів |
| Побудова MCP сервера (ч.1) | 3 | Архітектура: Host/Client/Server, Data layer, Transport layer, примітиви, lifecycle |
| Побудова MCP сервера (ч.2) | 4+ | Transport layer детально, stdio, HTTP, створення сервера |
| ... | ... | розширювати за мірою роботи |

**Лекція 1:** перша H1 «Введення до MCP» — наратив проблема/наслідки/рішення/схема/переваги/екосистема. Далі: Практичні сценарії → Висновки. **Не додавати** H1 «Що таке MCP» та «Архітектура MCP» — ці теми в наступних лекціях за планом курсу.

## Після публікації

1. Оновити `konspekt` ID у `content/notion-ids.json`, якщо створено нову сторінку.
2. **Обовʼязково:** створити підсторінки лекції — **Відео**, **Конспект**, **Quiz** (структура як у Лекції 1).
3. **Обовʼязково:** заповнити «Відео» — 3 toggle-секції H3 (Google doc, опис, презентація). Див. `.cursor/rules/mcp-course.mdc`.
4. Повідомити користувача URL сторінки та запропонувати `/generate-quiz` і `/notion-to-presentation`.

## Чеклист якості

- [ ] Лекція має підсторінки: Відео, Конспект, Quiz
- [ ] «Відео» містить 3 toggle-секції H3 (не порожня)
- [ ] Кожна глава — H1 з `{toggle="true"}`
- [ ] Немає заголовків глибше H3
- [ ] Є блок `# Висновки {toggle="true"}` — загальний підсумок заняття (не детальний flow)
- [ ] Bold не зловживається
- [ ] Є приклади коду або практичні кроки, де доречно
- [ ] Текст українською

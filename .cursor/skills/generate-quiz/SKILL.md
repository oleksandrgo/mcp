# Генератор Quiz MCP → Notion

## Мета

Створити або оновити сторінку **«Quiz»** з **5 питаннями** на основі конспекту лекції.

## Передумова

Конспект лекції **вже існує** в Notion. Якщо ні — спочатку виконати skill `generate-konspekt`.

## Вхід

- Номер лекції (наприклад, «Лекція 1»), або
- URL/ID сторінки «Конспект» в Notion.

ID — у [`content/notion-ids.json`](../../content/notion-ids.json).

## Вихід

Сторінка «Quiz» sibling під тією ж лекцією (оновлення через `notion-update-page`).

## Формат (Hillel MAX)

```
**1. Текст питання?**
A) Варіант A
B) Варіант B
C) Варіант C
D) Варіант D
**Правильна відповідь:** B
---
**2. ...**
...
**5. ...**
```

Альтернативний формат (теж допустимий):

```
### Тест 1:
**Питання:** ...
1. ...
2. ...
3. ...
4. ...
**Правильна відповідь:** 2. ...
---
```

## Правила

- Рівно **5 питань** — не більше, не менше.
- **4 варіанти** (A–D або 1–4) на кожне питання.
- Покриття: усі H1-блоки конспекту пропорційно (крім «Висновки» — max 1 питання).
- Складність:
  - 60% — базові (визначення, терміни)
  - 30% — прикладні (сценарії використання)
  - 10% — «чому» (розуміння архітектури)
- Не копіювати формулювання з конспекту дословно.
- Одна правильна відповідь; distractors мають бути правдоподібними.
- Мова: українська.
- **Заборонено:** питання про зовнішню документацію, workspace, meta-генерацію. Див. `.cursor/rules/mcp-course.mdc`.

## Алгоритм

1. Отримати конспект через `notion-fetch` (ID з `notion-ids.json`).
2. Витягнути H1/H2 та ключові терміни.
3. Згенерувати **5** питань.
4. Опублікувати на сторінку Quiz через `notion-update-page`.
